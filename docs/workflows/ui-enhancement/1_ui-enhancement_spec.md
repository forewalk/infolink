## 1. 개요

### 1.1 목적
Infolink 애플리케이션의 사용자 경험을 개선하기 위해 다국어 지원(i18n), 다크모드, 게시판 기능을 추가한다.

### 1.2 배경
- 글로벌 사용자를 위한 한글/영어 지원 필요
- 사용자 눈의 피로도 감소를 위한 다크모드 지원
- 사용자 간 커뮤니케이션을 위한 게시판 기능 필요

### 1.3 범위

**포함:**
- i18n: 한글/영어 언어 전환 기능
- 다크모드: 라이트/다크 테마 전환
- 게시판: 게시글 CRUD, 메인페이지 게시판 버튼

**제외:**
- 다른 언어 지원 (일본어, 중국어 등)
- 댓글 기능 (향후 추가 예정)
- 게시판 첨부파일 기능

---

## 2. 요구사항

### 2.1 기능 요구사항

#### 필수 기능 (Must Have)

**i18n:**
1. 한글/영어 언어 전환 버튼
2. 모든 UI 텍스트 다국어 지원
3. 언어 설정 localStorage 저장 (새로고침 유지)

**다크모드:**
1. 라이트/다크 테마 전환 토글
2. 시스템 설정 감지 (prefers-color-scheme)
3. 테마 설정 localStorage 저장

**게시판:**
1. 게시글 목록 조회
2. 게시글 작성 (제목, 내용)
3. 게시글 상세 보기
4. 게시글 수정/삭제 (작성자만)
5. 메인페이지 하단 게시판 버튼

#### 선택 기능 (Should Have)
1. 언어/테마 전환 애니메이션
2. 게시글 검색

#### 향후 고려사항 (Nice to Have)
1. 추가 언어 지원
2. 댓글 기능
3. 게시판 카테고리

### 2.2 비기능 요구사항

#### 성능
- 언어/테마 전환: 즉시 (깜빡임 없음)
- 게시판 목록 로딩: 500ms 이내

#### 보안
- 게시글 수정/삭제: 작성자 본인만 가능 (JWT 인증)
- 게시판은 로그인 성공한 사용자만 접근 가능, 로그인 성공하지 못한 경우, 로그인하러 가기 버튼 출력

## 3. 사용자 시나리오

### 3.1 주요 사용자
- 로그인 사용자: 모든 기능 사용 가능
- 비회원: 게시글 조회만 가능

### 3.2 사용 시나리오

#### 시나리오 1: 언어 변경
**사전 조건:** 사용자가 앱에 접속함

**실행 단계:**
1. 헤더의 언어 전환 버튼 클릭
2. 드롭다운에서 원하는 언어 선택

**기대 결과:**
- 모든 UI 텍스트가 선택한 언어로 변경
- 새로고침 후에도 유지

---

#### 시나리오 2: 다크모드 전환
**사전 조건:** 사용자가 앱에 접속함

**실행 단계:**
1. 헤더의 테마 토글 버튼 클릭

**기대 결과:**
- 화면이 다크/라이트 테마로 전환
- 새로고침 후에도 유지

---

#### 시나리오 3: 게시판 사용
**사전 조건:** 로그인한 사용자

**실행 단계:**
1. 메인페이지 하단 "게시판" 버튼 클릭
2. 게시판 페이지로 이동
3. "글쓰기" 버튼 클릭
4. 제목, 내용 입력 후 "저장"

**기대 결과:**
- 게시글이 목록에 표시됨

## 4. 데이터 요구사항

### 4.1 필요한 데이터 (게시판)

| 데이터 항목 | 타입 | 필수 여부 | 설명 |
|------------|------|----------|------|
| id | BIGINT | Y | PK |
| user_id | BIGINT | Y | 작성자 FK |
| title | VARCHAR(200) | Y | 게시글 제목 |
| content | TEXT | Y | 게시글 내용 |
| view_count | INT | Y | 조회수 (기본값 0) |
| created_at | TIMESTAMP | Y | 작성일시 |
| updated_at | TIMESTAMP | Y | 수정일시 |
| deleted_at | TIMESTAMP | N | 삭제일시 (soft delete) |

### 4.2 데이터 생성/수정/삭제
- 생성: 로그인 사용자가 게시글 작성
- 수정: 작성자가 게시글 수정
- 삭제: 작성자가 게시글 삭제 (soft delete)
- 조회: 모든 사용자

## 5. API 요구사항

### 5.1 게시판 API

#### API 1: 게시글 목록 조회
- **Method:** GET
- **Path:** /api/v1/boards
- **설명:** 게시글 목록 페이징 조회
- **요청 파라미터:** page, limit
- **응답:**
  ```json
  {
    "items": [...],
    "total": 100,
    "page": 1,
    "limit": 20
  }
  ```

#### API 2: 게시글 상세 조회
- **Method:** GET
- **Path:** /api/v1/boards/{id}

#### API 3: 게시글 작성
- **Method:** POST
- **Path:** /api/v1/boards
- **요청:**
  ```json
  {
    "title": "제목",
    "content": "내용"
  }
  ```

#### API 4: 게시글 수정
- **Method:** PUT
- **Path:** /api/v1/boards/{id}

#### API 5: 게시글 삭제
- **Method:** DELETE
- **Path:** /api/v1/boards/{id}
